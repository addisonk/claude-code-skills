# SVG Icon Audit Checklist

Use this checklist when reviewing AI-generated SVG icons or validating icon quality.

---

## 1. Structure Check (Do First)

- [ ] Root element is `<svg>` with correct namespace
- [ ] `viewBox="0 0 24 24"` present
- [ ] `width="24"` and `height="24"` present
- [ ] `fill="none"` on root SVG
- [ ] `stroke="currentColor"` on root SVG
- [ ] `stroke-width="2"` on root SVG
- [ ] `stroke-linecap="round"` on root SVG
- [ ] `stroke-linejoin="round"` on root SVG
- [ ] Only allowed elements used (path, circle, rect, line, polyline, polygon, ellipse)
- [ ] No `<g>`, `<defs>`, `<use>`, `<filter>`, `<text>` elements
- [ ] No `transform` attributes anywhere
- [ ] No `style`, `class`, or `id` attributes on children
- [ ] No explicit `fill`/`stroke`/`stroke-width` on child elements
- [ ] No `opacity` on elements
- [ ] All elements are direct children of `<svg>` (flat structure)

---

## 2. Design Rules Check

### Canvas & Padding

---

## 3. Visual Quality Check

### Optical Volume
- [ ] Icon doesn't appear significantly heavier/lighter than reference icons
- [ ] Blur test: blurred icon matches weight of blurred 24px circle
- [ ] Consistent density with similar-complexity icons

### Visual Centering
- [ ] Symmetric icons centered at (12,12)
- [ ] Asymmetric icons offset toward center of gravity
- [ ] Icon "feels" centered, not just bounding-box centered

### Smooth Curves
- [ ] No visible kinks or abrupt direction changes
- [ ] Adjacent curves have matching tangent angles
- [ ] Arc commands use appropriate radii

### Pixel Alignment
- [ ] Key coordinates on whole pixels (preferred)
- [ ] Circle/arc centers on whole pixels
- [ ] Line endpoints on whole pixels
- [ ] Fractional coords acceptable for organic curves only

---

## 4. Element Selection Check

- [ ] `<circle>` used for actual circles (not arc-paths)
- [ ] `<rect>` used for axis-aligned rectangles (not path-rectangles)
- [ ] Single `<path>` used when shape can be one continuous stroke
- [ ] Elements split only when parts are conceptually distinct
- [ ] No redundant elements (merged where possible)

---

## 5. Naming Check (if applicable)

- [ ] Lowercase kebab-case format
- [ ] Named for visual content, not function
- [ ] International English (not locale-specific)
- [ ] Group variants use `<group>-<variant>` format
- [ ] Modifiers as suffix: `[element]-[modifier]`
- [ ] No numerals (unless depicting a number)
- [ ] Multiple elements ordered by decreasing size in name

---

## 6. AI-Generated Icon Specific Checks

When reviewing icons generated by AI models (IconShop, etc.):

### Common AI Issues
- [ ] **No embedded styles** â€” AI often adds `<style>` blocks
- [ ] **No classes** â€” AI often adds class names for styling
- [ ] **No transforms** â€” AI often uses translate/rotate instead of absolute coords
- [ ] **No groups** â€” AI often wraps elements in `<g>`
- [ ] **Correct stroke width** â€” AI may use 1px, 1.5px, or variable widths
- [ ] **No fills on elements** â€” AI may add explicit fills for solid areas
- [ ] **Correct viewBox** â€” AI may use 16x16, 20x20, or 32x32
- [ ] **No gradients** â€” AI may add gradient definitions
- [ ] **No filter effects** â€” AI may add drop-shadow, blur, etc.
- [ ] **Round caps/joins** â€” AI may default to butt/miter

### Sanitization Steps for AI Output
1. Strip all `<defs>`, `<style>`, `<g>` wrappers
2. Remove all `class`, `id`, `style`, `transform` attributes
3. Remove explicit `fill`/`stroke`/`stroke-width` from children
4. Verify/set root SVG attributes to Lucide standard
5. Check viewBox is `0 0 24 24` (rescale if different)
6. Validate all remaining elements are in the allowed set
7. Check coordinates respect 1px padding rule
